1 How did I made the MIT License Download modal?

Curious, aren't you?

The MIT License Model is the one who appears when you click on the "DOWNLOAD SCREEN DEMO" button at the top. There is some active features on it, like a box-style text panel and a "I understand" checkbox. And now I'll show you how to make the same.

1. textcolor: 'white', for put the text in a white color, well.

2. bgcolor: 'rgb(23, 123, 175)', for change the background color. You can use it instead of "style" option, if you wish.

3. textalign: 'left', because the default is "center".

4. textborder: 'white', for make a box-style. The default for it is "none".

5. close: true, for show the Close button on the top right of the modal.

6. height: 350, since the 200px default isn't enough for a good reading.

7. gradient: {from,to}, just because simple gradients with slight alphas are beautiful.

8. check: "I understood, can I download now?", the most important line. That code line put an checkbox with its relative label. No button would work while that checkbox isn't active.

I wouldn't put all the text here, because it's giant... and, for security reasons, I wouldn't put the full link of the zipped plugin. Feel free for to change anything, because... well, it's MIT License.

	
			$.jMetro(
				"MIT License",
				"The long text goes here...",
				{
					height: '350',
					gradient: {
						from: 'rgba(2,127,185,1)',
						to: 'rgba(11,128,183,.9)'
					},
					textalign: 'left',
					textcolor: 'white',
					textborder: 'white',
					check: "I understood, can I download now?",
					close: true,
					buttons: [{
						text: "Download",
						link: " path/to/plugin.rar",
						target: "same"
					}]
				}
			);

2 Options Item

There is plenty of options that you can use, for change your modal experience. Some of them are default, but you can set/hide then normally.

1. blocker: true the grey area of the "rest" of the modal. It doesn't allow the user to click on the page until he hides the modal.
For set it off, put blocker: false
2. fadeIn: 200 the speed of fade in. Default is 200ms. If you want to remove that animation, put fadeIn: false.
3. fadeOut: 200 the speed of fade in. Default is 200ms. If you want to remove that animation, put fadeOut: false.
4. style: "none" the style of the modal. Default is "none". There is 5 styles: none, error, success, dark and white.
5. opacity: 1 the opacity of the modal. Default is 1.
6. height: 200 the height of the modal. Default is 200.
7. bgcolor: "white" the color of the modal (a hexadecimal number). Setting it will override the style you choose before. Default is '#027FB9'.
8. gradient: {top;from} adds a simple top-down gradient with RGBA() and alpha support. If the browser doesn't support it, the color you put in "from" will be the default color.
9. textalign: "center" the alignment of the text area. It can be "left", "right" and "center". Default is "center".
10. textcolor: "white" the font color of the text area. Setting it will override the style you choose before. Default is white.
11. textborder: "none" the border color of the text area. Normally used for "Read This" alerts. If you don't want borders, do not set this parameter.
12. textshadow: false the text shadow in the modal. Default is false (no shadow). If you want to put shadow, use a hexadecimal color (textshadow: #3323ee).
13. shadowcolor: #000 the color of the blocker. Default is black. If you want to unable the blocker, put blocker: false.
14. shadowopacity: .4 the opacity of the blocker. Default is 0.4 (from 0 to 1). If you want to unable the blocker, put blocker: false.
15. icon: http://.... a link for an image, for be used as modal' icon. It resizes automatically to 32 x 32.
16. image: http://.... a link for an image, for be used as modal' background. That image will repeat infinitely.
17. check: "Are you sure?" a checkbox to be used as a alert confirm. Normally used in "Read This" alerts. While the user doesn't check the box, the modal wouldn't disappear, and an common alert will pop-up. Use it with the "close" button.
18. close: false creates a close button on the top right of the modal. Clicking on it would hide it, like a "cancel", but without firing any event.

3 More samples

This sample show how to create a custom alert, including a new blocker color.

	
$.jMetro(
	"Alert!",
	"Something is wrong... insert error here.",
	{
		icon: 'http://goo.gl/aQJHgu',
		style: "error",
		shadowcolor: '#850000'
	}
);

Bit complex: how to create a "Yes or No" modal, showing another jMetro depending on the selected button.

	
$.jMetro(
	"You need to choose!",
	"Someone wants to eat a piece of cake. Do you allow it?",
	{
		buttons: [{
			text: "YES",
			click: function(){
				$.jMetro(
					"Nham nham nham",
					"He ate the cake, and he liked it~",
					{
						style: "success",
						shadowcolor: '#3a6f00'
					}
				);		
			}		
		},{
			text: "NO",
			click: function(){
				$.jMetro(
					"OMG WHY",
					"I wanted that cake so bad DXX",
					{
						style: "error",
						shadowcolor: '#850000'
					}
				);			
			}		
		}]
	}
);
